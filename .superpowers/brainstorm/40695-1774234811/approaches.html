Architecture Approach

All three use Next.js + Vercel. The difference is in the database and auth layer.

A

Next.js + Prisma + PostgreSQL (Vercel Postgres) + NextAuth

Recommended

Pros

  • Vercel Postgres is fully managed, zero DevOps
  • Prisma gives you type-safe queries and easy migrations
  • NextAuth has built-in Discord provider
  • All services on one platform (Vercel) — simple billing and deployment
  • Generous free tier for small leagues

Cons

  • Vercel Postgres has storage limits on free tier
  • Vendor lock-in to Vercel's database
B

Next.js + Prisma + Supabase (PostgreSQL) + NextAuth

Pros

  • Supabase has a very generous free tier (500MB)
  • Built-in dashboard for viewing/editing data
  • Can use Supabase Auth instead of NextAuth if preferred
  • More portable — not locked to Vercel for DB

Cons

  • Two platforms to manage (Vercel + Supabase)
  • Slightly more complex setup
  • Cold starts on free tier can pause after inactivity
C

Next.js + Drizzle ORM + Turso (SQLite) + NextAuth

Pros

  • Turso is extremely fast (edge SQLite)
  • Very generous free tier (9GB)
  • Drizzle is lightweight and fast

Cons

  • SQLite is less common — fewer tutorials/examples
  • Three platforms (Vercel + Turso + Discord)
  • Drizzle has a smaller ecosystem than Prisma
